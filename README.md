# HANDAVIS ⛨

## Meaning and Vision

**HANDAVIS** is derived from **“HANDA”** (Filipino for "Prepared") and **“VISAYAS”**, symbolizing its focus on **preparing and protecting communities across the Visayas region**. The name reflects our mission:

> **“Empowering communities to be ready, resilient, and responsive during emergencies.”**

This branding underscores the platform's commitment to **real-time, life-saving coordination** for the entire Western Visayas.

---

## A Community-Based, Real-Time Emergency Response Platform for Western Visayas

HANDAVIS is a **comprehensive, multi-portal emergency response system** designed to enhance disaster preparedness, response coordination, and public safety across **Western Visayas**. It integrates **real-time tracking, live news updates, live weather forecasts, one-tap hazard reporting, SOS alerts, AI-powered assistance, smart evacuation routing, and community circle monitoring** into a single, unified platform.

The system is engineered to reduce response time, improve situational awareness, and empower both citizens and authorities to act quickly and effectively during emergencies.

## 1. Objectives

HANDAVIS aims to:

* Provide **real-time situational awareness** during disasters
* Enable **fast and reliable emergency communication**
* Improve **coordination between civilians, responders, and local authorities**
* Support **data-driven evacuation and response decisions**
* Strengthen **community resilience** through connected systems
* Empower users with **AI guidance and actionable insights**

---

## 2. System Architecture Overview

HANDAVIS follows a **multi-role, web-based architecture** composed of four primary portals:

* User Portal
* Barangay Staff Portal
* Responder Portal
* Admin Portal

Each portal has **role-based access control (RBAC)** to ensure secure and appropriate system usage.

---

## 3. System Portals and Features

### 3.1 User Portal

**Key Functions:**

* **SOS alerts** with real-time location
* **One-tap hazard reporting**
* **Real-time map view** with active hazards and safe zones
* **Live weather forecasts and news updates**
* **Smart evacuation route recommendations**
* **Circle system** to monitor trusted contacts and their safety status
* **HANDAm AI assistant** for disaster guidance and emergency instructions

---

### 3.2 Barangay Staff Portal

**Key Functions:**

* Monitor **localized incidents and reports**
* Validate and manage **hazard submissions**
* Track **SOS alerts within their barangay**
* Coordinate community response
* Update incident statuses
* Support **real-time communication with users and responders**

---

### 3.3 Responder Portal

**Key Functions:**

* Receive **real-time SOS alerts**
* View **exact user locations**
* Navigate using **optimized evacuation routes**
* Access **live hazard maps**
* Update response progress and status
* Integrate **AI guidance for disaster management**

---

### 3.4 Admin Portal

**Key Functions:**

* Manage **users, roles, and permissions**
* Monitor **all system activities and reports**
* Maintain **data integrity and system performance**
* Oversee platform operations and analytics
* Ensure **proper coordination among all portals**

---

## 4. Core System Features

* **Real-Time Tracking**: Continuous location updates for users and incidents
* **Real-Time News Updates**: Aggregated disaster-related information
* **Real-Time Weather Forecast**: Current conditions and alerts
* **One-Tap Hazard Reporting**: Instant reporting of floods, fires, storms, earthquakes/ashfall, roadblocks, and medical emergencies
* **SOS Emergency System**: Immediate alerts to circle, responders, and admin
* **Smart Evacuation Route Recommendation**: Optimized paths avoiding hazards
* **Circle Feature (Community Network)**: Monitor trusted contacts, receive/send alerts, share status updates
* **HANDAm AI Assistance**: Disaster-related queries, guidance, and actionable instructions

---

## 5. System Workflow

1. User logs into the system and enables location services
2. Dashboard displays:

   * Real-time map
   * Weather updates
   * News feed
3. In case of emergency:

   * User triggers SOS or reports hazard
4. System processes and:

   * Notifies circle members, responders, and admin
   * Updates map and system data in real time
   * Provides smart evacuation routes
   * Offers AI guidance via HANDAm
5. Responders act using provided location and route data
6. Incident is resolved and status is updated

## 6. Problem Statement

Current emergency response systems face several challenges:

* Delayed response times
* Lack of real-time, centralized information
* Inefficient coordination between stakeholders
* Unsafe or unclear evacuation planning

* Ensuring **community safety through circle monitoring**
